1. HOW I BUILT MY FIRST WEBSITE USING HTML.

INTRODUCTION

When I first started learning web development, I wanted to build something more than just a single-page website. I wanted to create a multi-page website where each section had its own dedicated page, just like real websites do.My goal was to build a simple yet well-structured website with the following pages:

  • Home - The main landing page
  • Projects - A section to showcase my work
  • Articles - A blog-like section for sharing knowledge
  • Contact - A way for visitors to reach me

Since I was focusing only on HTML structure (without CSS or JavaScript), I needed to ensure my code was clean, semantic, and easy to style later. This article documents the step-by-step process I followed, including lessons learned and mistakes I made along the way.

Step 1 : Planning the Website Structure

Before jumping into coding, I took some time to plan my website’s structure. A well-planned structure makes a website easier to navigate and update in the future.

I decided on the following layout for each page:

  1. A Navigation Bar (Same on all pages)
  2. Main Content Section (Different for each page)
  3. A Footer

Lesson Learned: Planning first saves time later. Without a clear plan, I would have had to rewrite my code multiple times.

Step 2: Creating Multiple HTML Pages

With the structure planned, I created separate HTML files:

  • index.html (Home)
  • projects.html (Projects)
  • articles.html (Articles)
  • contact.html (Contact)

Each file would contain the same navigation bar to ensure consistency.

Lesson Learned: Consistent elements improve user experience. Keeping the nav bar same across all pages makes navigation smooth.

Step 3: Building a Consistent Navigation Bar

One of the first things I worked on was the navigation bar. Since this element appears on every page, I made sure it was:

  • Simple Just links, no dropdowns or advanced styling
  • Semantic Wrapped in a <nav> element

Lesson Learned: Since I am using HTML only, I can't make the navigation bar to automatically update in all pages when there is a change.

Step 4: Structuring Each Page with Semantic HTML

Instead of using only <div> elements, I used semantic HTML to make my code more readable and accessible.

For example, my homepage (index.html) was structured like this:

  • <header> - contains the site title and navigation bar
  • <main> - holds the main content of the page
  • <footer> - includes copyright and additional links

On the Projects page (projects.html), I used:

  • <section> - to group different project categories
  • <article> - to define individual project descriptions

For the Articles page (articles.html), I used:

  • <article> - for individual blog posts

Lesson Learned: Using semantic tags improves SEO and accessibility. Search engines and screen readers can better understand the page structure.

Step 5: Linking the Pages Together

To make navigation seamless, I added links to each page inside the nav bar.

I used relative paths so the links worked correctly:

  • href="index.html"=Links to Home
  • href="projects.html"=Links to Projects
  • href="articles.html"=Links to Articles
  • href="contact.html"=Links to Contact
    •

Step 6: Adding SEO Meta Tags

Although I wasn't focusing on styling, I wanted my website to be search engine-friendly.

So, I added the following meta tags in the <head> section of my homepage: 

    
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <meta name="description" content="A Simple HTML Portfolio Website showcasing the works of Front-End Developer Ozobia Alfred">
      <meta name="keywords" content="frontend developer, web developer, portfolio, Ozobia Alfred, front-end, javascript, html, css, react, node, tailwind, bootstrap">
      <meta name="author" content="Ozobia Alfred">

Corresponding SEO meta tags were also added to the subsequent pages

Lesson Learned: Good SEO starts with structure, not just content. These tags help search engines understand what the page is about.

Step 7: Adding a Footer with Useful Links

Just like the navigation bar, I wanted a consistent footer on all pages. I included:

  • Copyright Notice (© all rights reserved 2025 )
  • Links to Social Profiles

Lesson Learned: A footer makes your site look more complete and professional. It also helps with navigation, especially for long pages.

Conclusion: What I Learned

Building this multi-page website was a great learning experience! Here are my biggest takeaways:

  • Planning is key-Sketching the structure first made development smoother.
  • Consistency matters-Having the same navigation bar and footer made the site feel more professional.
  • Semantic HTML is important-Using <header>, <nav>, <article>, and <section> made the code more readable.
  • SEO starts with good structure-Simple meta tags help search engines understand the page better.

What's Next?

Now that my site is structured, the next step is to:

  • Add CSS for styling
  • Improve responsiveness for mobile devices
  • Explore JavaScript for interactivity

If you're a beginner looking to build your first multi-page website , I highly recommend trying this! It's a great way to understand how real websites are structured > .
